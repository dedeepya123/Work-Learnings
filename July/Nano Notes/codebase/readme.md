## For each important component I encounter, ask:
- What does this module do?
- Why does it exist?
- Where is it called from?
- What are its inputs and outputs?
- How does it fit into the overall pipeline?
- What assumptions is it making?
- What questions do I still have?

## Every Friday
ASK 
- What did I learn this week?
- Which part of the pipeline is still unclear?
- Which architecture concept explains something I saw in the code?
- Which questions should I ask my teammates next week

Suppose, I notice a function dealing with grouped-query attention. Evening Read about GQA in the Gemma architecture and understand why it exists.

Next day Go back to the code and see if it makes more sense.

That's a feedback loop:
``` text
Code
   ↓
Question
   ↓
Paper
   ↓
Understanding
   ↓
Back to code
```
